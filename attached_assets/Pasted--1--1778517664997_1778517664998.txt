현재 카카오톡 봇 기반 게임 서비스인 “초능력자 봇”에 닉네임 변경권 기능을 추가하려고 합니다.

이 프로젝트는 1인 개발 프로젝트입니다.
따라서 관리자 검수, 수동 승인, 검수 대기, 사람이 직접 확인하는 운영 방식은 사용할 수 없습니다.
닉네임 변경 요청이 들어오면 자동 필터만으로 통과/차단을 판단해야 합니다.

기존 닉네임 금지어 필터 코드는 TypeScript로 작성되어 있으며, 현재 구조는 대략 다음과 같습니다.

1. 닉네임 길이 검사
2. 공백 검사
3. 퍼지 정규식으로 글자·숫자 삽입 우회 차단
4. 정규화 후 금지 부분문자열 포함 여부 검사
5. 한글 자모 분해 후 Levenshtein 유사도 기반 검사
6. 현재 자모 유사도 기준은 80%입니다.

기존 방향 자체는 유지하되, 대형 게임/플랫폼에서 사용하는 닉네임 필터 방식처럼 다음 요소를 반영해서 더 안정적인 자동 차단 시스템으로 리팩토링해주세요.

# 구현 요구사항

## 1. normalize 함수 강화

기존 normalize 함수보다 더 강하게 정규화해주세요.

반드시 포함할 기능:

* `NFKC` 정규화
* lowercase 처리
* zero-width 문자 제거
* 공백 제거
* 숫자 제거
* 특수문자 제거
* 연속 동일 문자 1자로 축소

예시:

```ts
function normalize(s: string): string {
  return s
    .normalize('NFKC')
    .toLowerCase()
    .replace(/[\u200B-\u200D\uFEFF]/g, '')
    .replace(/[\s\d!"#$%&'()*+,\-./:;<=>?@[\\\]^_`{|}~]/g, '')
    .replace(/(.)\1+/g, '$1');
}
```

필요하다면 더 안정적으로 수정해도 됩니다.

## 2. 허용 문자 검사 추가

닉네임은 한글, 한글 자모, 영문, 숫자만 허용합니다.

예시:

```ts
const nickNfkc = nick.normalize('NFKC');

if (!/^[가-힣ㄱ-ㅎㅏ-ㅣa-zA-Z0-9]+$/.test(nickNfkc)) {
  return { ok: false, reason: '닉네임은 한글, 영문, 숫자만 사용할 수 있어요.' };
}
```

이 검사는 길이 검사 직후에 배치해주세요.

## 3. 운영자/봇명/시스템 사칭어는 별도 하드차단

욕설과 사칭은 성격이 다릅니다.

욕설은 커뮤니티 분위기 문제지만, 운영자/관리자/공식 사칭은 서비스 신뢰도 문제입니다.
따라서 아래 단어들은 일반 금지어와 분리해서 하드차단해주세요.

```ts
const RESERVED_WORDS = [
  '운영자',
  '운영진',
  '관리자',
  '어드민',
  'admin',
  'administrator',
  'manager',
  'gm',
  'official',
  'system',
  'notice',
  'support',
  '공지',
  '공식',
  '시스템',
  '고객센터',
  '초능력자',
  '초능력자봇',
];
```

검사 방식:

* normalize된 닉네임에 normalize된 예약어가 포함되어 있으면 차단
* 대소문자 변형도 차단
* 전각문자 변형도 차단
* `adm1n`, `Ａｄｍｉｎ` 같은 우회도 최대한 차단

사용자에게는 구체적인 매칭 단어를 알려주지 말고 아래 공통 문구를 반환해주세요.

```ts
return { ok: false, reason: '사용할 수 없는 닉네임이에요.' };
```

## 6. 초성 욕설 필터 추가

한글 음절에서 초성만 추출해서 초성 욕설을 검사하는 함수를 추가해주세요.

예시:

```ts
function getChosung(s: string): string {
  return Array.from(s).map(ch => {
    const code = ch.charCodeAt(0);
    if (code >= 0xAC00 && code <= 0xD7A3) {
      const off = code - 0xAC00;
      return CHO_TABLE[Math.floor(off / 588)];
    }
    return ch;
  }).join('');
}
```

차단 대상 예시:

```ts
const BANNED_CHOSUNG = [
  'ㅅㅂ',
  'ㅆㅂ',
  'ㅂㅅ',
  'ㅈㄴ',
  'ㅁㅊ',
  'ㅈㄹ',
  'ㅅㄲ',
  'ㄱㅅㄲ',
];
```

검사 함수:

```ts
function hasBannedChosung(nick: string): boolean {
  const cho = getChosung(normalize(nick));
  return BANNED_CHOSUNG.some(w => cho.includes(w));
}
```

## 7. 자모 유사도 threshold를 카테고리별로 분리

현재는 자모 유사도 기준이 0.80 하나로 되어 있습니다.
이를 단어별 또는 카테고리별 threshold로 바꿔주세요.

추천 기준:

* 욕설류: 0.75 ~ 0.78
* 정치인/논쟁성 인물: 0.82
* 운영자/운영진/관리자/봇명 사칭: 0.85
* 일반 금지어: 0.80

예시 구조:

```ts
  { word: '씹새끼', jamo: decomposeJamo('씹새끼'), threshold: 0.75, category: 'profanity' },
  { word: '병신새끼', jamo: decomposeJamo('병신새끼'), threshold: 0.78, category: 'profanity' },

  { word: '윤석열', jamo: decomposeJamo('윤석열'), threshold: 0.82, category: 'political' },
  { word: '문재인', jamo: decomposeJamo('문재인'), threshold: 0.82, category: 'political' },
  { word: '노무현', jamo: decomposeJamo('노무현'), threshold: 0.82, category: 'political' },

  { word: '운영자', jamo: decomposeJamo('운영자'), threshold: 0.85, category: 'reserved' },
  { word: '관리자', jamo: decomposeJamo('관리자'), threshold: 0.85, category: 'reserved' },
  { word: '초능력자', jamo: decomposeJamo('초능력자'), threshold: 0.85, category: 'bot' },

```

검사 예시:

```ts
for (const { jamo: bannedJamo, threshold } of JAMO_FUZZY_TARGETS) {
```

## 9. 전화번호/개인정보성 숫자 패턴 차단

닉네임에 숫자는 허용하지만, 전화번호처럼 보이는 숫자열은 차단하고 싶습니다.

다음 패턴은 차단해주세요.

* 숫자 8자리 이상 연속
* `010`으로 시작하는 숫자 조합
* 생년월일처럼 보이는 6~8자리 숫자는 가능하면 경고보다는 차단

예시:

```ts
function hasPersonalInfoPattern(nick: string): boolean {
  const digits = nick.replace(/\D/g, '');

  if (/\d{8,}/.test(digits)) return true;
  if (/010\d{7,8}/.test(digits)) return true;

  return false;
}
```

검사 시 차단 문구는 공통 문구를 사용해주세요.

```ts
return { ok: false, reason: '사용할 수 없는 닉네임이에요.' };
```

# 닉네임 변경권 소모 정책

닉네임 변경권은 반드시 닉네임 검사 성공 후에만 소모되어야 합니다.

원하는 흐름:

```ts
const result = checkNickname(newNickname);

if (!result.ok) {
  return {
    success: false,
    message: result.reason,
    consumeTicket: false,
  };
}

// 여기서부터 변경권 차감
// 1. 유저가 닉네임 변경권을 가지고 있는지 확인
// 2. 변경권 1개 차감
// 3. 닉네임 변경
// 4. 변경 완료 메시지 반환
```

절대 아래 흐름으로 작성하지 마세요.

```ts
// 잘못된 흐름
// 1. 변경권 먼저 차감
// 2. 닉네임 검사
// 3. 실패 시 변경권이 이미 사라짐
```

차단된 닉네임에는 닉네임 변경권이 소모되면 안 됩니다.

# 차단 로그 저장 구조 제안

수동 검수나 신고 시스템은 사용하지 않지만, 나중에 어떤 닉네임이 자동 차단되었는지 확인할 수 있도록 차단 로그 구조를 제안해주세요.

예시:

```ts
type NicknameBlockLog = {
  userId: string;
  attemptedNick: string;
  normalizedNick: string;
  reason: string;
  matchedRule?: string;
  category?: 'profanity' | 'sexual' | 'political' | 'reserved' | 'bot' | 'format' | 'personalInfo';
  createdAt: Date;
};
```

단, 사용자에게는 matchedRule이나 category를 노출하지 마세요.
관리용 로그로만 사용합니다.

그리고 이 로그는 신고 시스템이 아닙니다.
사용자의 신고를 받거나, 신고 횟수를 누적하거나, 신고를 기반으로 닉네임을 변경하는 기능은 만들지 마세요.
