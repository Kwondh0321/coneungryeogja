초능력자 카카오톡 봇의 배틀 및 랭킹 시스템을 개편하고자 합니다.

현재 초능력자 봇은 포인트(P), 초성퀴즈, 훈민정음, 자모연성, 콤보, 유물, 배틀, 웹 랭킹 시스템으로 구성되어 있습니다.

현재 고민 중인 핵심 문제는 다음과 같습니다.

1. 기존 포인트 랭킹은 현재 보유 포인트만 기준으로 하므로, 유물 뽑기·강화·합성·레벨업에 포인트를 많이 투자한 유저가 랭킹에서 크게 밀리는 문제가 있습니다.

2. 기존 유물/전투력 시스템은 주 유물 1개만 중심으로 전투력을 계산하는 문제가 있습니다. 예를 들어 S등급 +2강 유물 1개만 가진 유저가, A등급 +10강 유물을 여러 개 가진 유저보다 무조건 강하게 취급되는 것은 불합리합니다.

3. 단순 유물 랭킹은 장기적으로 상향평준화될 가능성이 있습니다. 시간이 지나면 상위 유저들이 모두 S등급, 고강화, 고레벨 유물을 갖게 되어 공동 1등이 많아질 수 있습니다.

4. 기존 포인트 랭킹의 장점은 실시간 변동성이 크다는 점입니다. 유저들이 포인트를 벌고, 쓰고, 배틀하고, 강화하면서 순위가 계속 움직입니다. 이 장점은 유지하고 싶습니다.

따라서 랭킹 시스템을 단순히 “포인트 랭킹 / 유물 랭킹 / 통합 랭킹”으로 고정하지 말고, 더 사용자 친화적이고 운영상 납득 가능한 구조로 재설계하고 싶습니다.

이번 개편에서 반드시 고려해야 할 방향은 다음과 같습니다.

━━━━━━━━━━━━━━━━━━━━
1. 유물 자산가치 개념 추가
━━━━━━━━━━━━━━━━━━━━

각 유물에는 “유물 자산가치”를 계산해야 합니다.

유물 자산가치 공식은 다음과 같습니다.

유물 자산가치 =
유물 기본가치
+ 강화 누적 비용
+ 레벨업 누적 비용
+ 합성 비용 반영분

구체적으로는 다음 기준을 사용합니다.

1) 유물 기본가치

유물 기본가치는 등급별 기준값을 사용합니다.
기존 유물 분해 기본 환급액을 기본가치로 사용해도 됩니다.

예시:

D등급: 630P
C등급: 2,530P
B등급: 10,100P
A등급: 37,970P
S등급: 126,570P

단, 이 값은 환경변수 또는 설정값으로 관리할 수 있게 해주세요.

2) 강화 누적 비용

해당 유물에 실제로 사용된 강화 비용의 누적합입니다.

중요:
강화 비용 절감 효과가 적용되었다면, 명목상 비용이 아니라 실제로 지불한 포인트를 기준으로 기록해야 합니다.

예시:
원래 강화 비용이 10,000P인데 강화 비용 절감 효과로 8,000P만 지불했다면,
강화 누적 비용에는 8,000P만 반영합니다.

3) 레벨업 누적 비용

해당 유물에 실제로 사용된 레벨업 비용의 누적합입니다.

마찬가지로 할인, 보너스, 비용 절감 효과가 있다면 실제 지불한 값을 기준으로 기록합니다.

4) 합성 비용 반영분

유물 합성으로 생성된 유물은 합성에 사용된 하위 유물들의 자산가치와 합성 수수료를 함께 승계해야 합니다.

예시:

D등급 유물 3개를 합성해서 C등급 유물을 만들 경우,

새 C등급 유물의 자산가치 =
합성에 사용된 D유물 3개의 자산가치 합
+ D→C 합성 수수료
+ C등급 기본가치 보정분

다만 중복 계산이 발생하지 않도록 주의해야 합니다.

추천 방식:

새 유물 자산가치 =
max(
  상위 등급 기본가치,
  재료 유물 자산가치 합 + 합성 수수료
)

또는 더 단순하게:

새 유물 자산가치 =
재료 유물 자산가치 합 + 합성 수수료

이 방식 중 하나를 선택하되, 코드 내부에서 일관되게 적용해주세요.

5) 자산가치 저장 방식

가능하다면 유물 테이블에 다음 컬럼을 추가하거나, 별도 로그 테이블을 활용해주세요.

- base_asset_value
- enhance_cost_total
- levelup_cost_total
- synthesis_inherited_value
- relic_asset_value

최종 relic_asset_value는 다음과 같이 계산됩니다.

relic_asset_value =
base_asset_value
+ enhance_cost_total
+ levelup_cost_total
+ synthesis_inherited_value

단, 실제 운영에서는 매번 전체 로그를 다시 계산하기보다,
강화/레벨업/합성 이벤트가 발생할 때마다 누적값을 업데이트하는 방식을 권장합니다.

━━━━━━━━━━━━━━━━━━━━
2. 실효 전투력 개념 추가
━━━━━━━━━━━━━━━━━━━━

기존 배틀 시스템은 주 유물 1개만 전투력에 반영하고 있습니다.
이를 다음 구조로 개편하고 싶습니다.

실효 전투력 =
메인 유물 전투력 × 100%
+ 보관함에서 실제 효과가 적용되는 유물들의 전투력 × 보관함 적용률

현재 보관함 적용률은 35.31%입니다.

즉,

실효 전투력 =
mainRelicPower
+ storageRelicPowerSum × 0.3531

단, 보관함 유물은 실제 게임 효과 적용 규칙과 동일하게 처리해야 합니다.

규칙:

1. 메인 유물은 100% 반영합니다.
2. 보관함 유물은 35.31%만 반영합니다.
3. 같은 효과 타입은 최대 3개까지만 반영합니다.
4. 보관함 효과 증폭 유물이 있다면 기존 게임 효과 계산 방식과 동일하게 반영합니다.
5. 실제 효과가 적용되지 않는 4번째 이후 중복 타입 유물은 전투력에도 반영하지 않습니다.
6. 단순히 소유만 하고 있고 보관함에 장착되지 않은 유물은 배틀 전투력에는 반영하지 않습니다.

중요:
배틀 전투력은 “전체 소유 유물” 기준이 아니라,
“실제로 장착되어 효과가 적용되는 유물” 기준이어야 합니다.

따라서 다음 두 값을 분리해주세요.

1. 유물 자산가치
- 소유한 모든 유물의 가치
- 유저가 유물에 투자한 총자산을 보여주는 값

2. 실효 전투력
- 메인 유물 + 실제 적용 중인 보관함 유물 기준
- 배틀과 전투력 랭킹에 사용하는 값

━━━━━━━━━━━━━━━━━━━━
3. 랭킹 시스템 개편안
━━━━━━━━━━━━━━━━━━━━

통합 랭킹을 단순히 “포인트 + 유물점수”로 계산하는 방식은 유저 입장에서 이해하기 어렵고, 가중치 논란이 생길 수 있습니다.

따라서 다음과 같은 랭킹 구조를 검토해주세요.

추천 랭킹 탭:

1. 🌟 초능력자 랭킹
2. 💎 보유 포인트 랭킹
3. 🏺 유물 자산 랭킹
4. ⚔️ 전투력 랭킹
5. 🏆 시즌/레이팅 랭킹

각 랭킹의 의미는 다음과 같습니다.

━━━━━━━━━━━━━━━━━━━━
3-1. 🌟 초능력자 랭킹
━━━━━━━━━━━━━━━━━━━━

초능력자 랭킹은 대표 랭킹입니다.

단순 통합 랭킹처럼 포인트와 유물점수를 임의 가중치로 더하는 방식이 아니라,
유저의 전체 성장 상태를 보여주는 대표 점수로 설계합니다.

1차 공식 후보:

초능력자 점수 =
현재 보유 포인트
+ 전체 유물 자산가치
+ 실효 전투력 보너스

여기서 전체 유물 자산가치는 유저가 소유한 모든 유물의 relic_asset_value 합입니다.

전체 유물 자산가치 =
Σ relic_asset_value

실효 전투력 보너스는 너무 크게 반영하지 말고 보조적 요소로만 넣습니다.

예시:

초능력자 점수 =
현재 보유 포인트
+ 전체 유물 자산가치
+ log 기반 전투력 보너스

전투력 보너스 예시:

battlePowerBonus =
K × log10(1 + effectiveBattlePower)

K는 환경변수로 관리합니다.

예시 환경변수:

RANKING_BATTLE_POWER_LOG_WEIGHT=100000

이 방식의 장점:
- 포인트를 보유한 유저도 인정됩니다.
- 유물에 포인트를 투자한 유저도 인정됩니다.
- 실제 전투력이 높은 유저도 약간의 보너스를 받습니다.
- 전투력 보너스를 로그로 처리하면 상위권 점수가 과도하게 폭발하지 않습니다.

중요:
초능력자 랭킹은 대표 랭킹이지만, 유저가 이해하기 쉬워야 합니다.
따라서 웹 화면에는 다음처럼 표시해주세요.

예시 표시:

🌟 초능력자 점수: 1,532,000점
보유 포인트: 420,000P
유물 자산: 980,000점
전투력 보너스: 132,000점

이렇게 세부 내역을 보여주면 유저가 자신의 순위를 납득할 수 있습니다.

━━━━━━━━━━━━━━━━━━━━
3-2. 💎 보유 포인트 랭킹
━━━━━━━━━━━━━━━━━━━━

기준:
현재 보유 포인트

DB 기준:
users.score

의미:
현재 가장 많은 포인트를 보유한 유저 순위입니다.

주의:
이 랭킹은 “누적 획득 포인트”가 아니라 “현재 보유 포인트” 기준임을 명확히 표시해주세요.

표시명은 “포인트 랭킹”보다 “보유 포인트 랭킹”을 권장합니다.

예시:

💎 보유 포인트 랭킹
현재 보유 중인 포인트 기준 순위입니다.
유물 강화, 합성, 뽑기 등에 포인트를 사용하면 순위가 변동될 수 있습니다.

━━━━━━━━━━━━━━━━━━━━
3-3. 🏺 유물 자산 랭킹
━━━━━━━━━━━━━━━━━━━━

기준:
유저가 소유한 모든 유물의 자산가치 합

공식:

유물 자산점수 =
Σ relic_asset_value

의미:
유저가 유물에 얼마나 많은 자원을 투자했는지 보여주는 랭킹입니다.

이 랭킹은 배틀 실력이나 실제 전투력과는 다릅니다.
소유 유물 전체의 자산 규모를 보여주는 랭킹입니다.

표시 예시:

🏺 유물 자산 랭킹
유물 자산: 1,240,000점
보유 유물: 18개
최고 유물: A등급 +13강 Lv.20

주의:
이 랭킹은 포인트를 유물에 투자한 유저가 인정받는 용도입니다.
단순 전투력 랭킹과 분리해야 합니다.

━━━━━━━━━━━━━━━━━━━━
3-4. ⚔️ 전투력 랭킹
━━━━━━━━━━━━━━━━━━━━

기준:
실효 전투력

공식:

실효 전투력 =
메인 유물 전투력
+ 실제 적용 보관함 유물 전투력 합 × 보관함 적용률

보관함 적용률 기본값:
0.3531

의미:
현재 실제 배틀에서 얼마나 강한지를 보여주는 랭킹입니다.

이 랭킹은 소유 유물 전체가 아니라, 실제 장착되어 효과가 적용되는 유물을 기준으로 합니다.

표시 예시:

⚔️ 전투력 랭킹
실효 전투력: 235,420
메인 유물 기여: 142,000
보관함 유물 기여: 93,420

━━━━━━━━━━━━━━━━━━━━
3-5. 🏆 시즌/레이팅 랭킹
━━━━━━━━━━━━━━━━━━━━

통합 랭킹의 대안으로, 장기적으로는 시즌/레이팅 랭킹을 도입하는 것을 검토합니다.

이 랭킹은 포인트나 유물 자산을 단순 합산하는 것이 아니라,
최근 활동성과 실제 경쟁 성과를 반영합니다.

후보 A: 시즌 활동 랭킹

기준:
일정 기간 동안의 활동 점수

예시 공식:

시즌 활동점수 =
시즌 획득 포인트
+ 출석 점수
+ 퀴즈 정답 점수
+ 훈민정음 MVP 점수
+ 배틀 승리 점수

예시 가중치:

출석 1회: 1,000점
초성퀴즈 정답: 100점
자모연성 정답: 100점
훈민정음 정답: 100점
훈민정음 MVP: 1,000점
배틀 승리: 500점

이 값들은 환경변수 또는 설정값으로 관리할 수 있게 해주세요.

시즌 주기:
주간 또는 월간

장점:
- 신규 유저도 시즌마다 도전할 수 있습니다.
- 고인물 고착화를 줄일 수 있습니다.
- 랭킹이 계속 움직입니다.

후보 B: 전투 레이팅 랭킹

기준:
배틀 결과에 따라 오르내리는 레이팅 점수

초기 레이팅:
1000점

레이팅 방식:
Elo 또는 간단한 MMR 방식을 사용합니다.

기본 아이디어:
- 강한 상대를 이기면 많이 오릅니다.
- 약한 상대를 이기면 조금 오릅니다.
- 강한 상대에게 지면 조금 떨어집니다.
- 약한 상대에게 지면 많이 떨어집니다.

예시:

expectedWinRate =
1 / (1 + 10 ^ ((opponentRating - myRating) / 400))

newRating =
oldRating + K × (actualResult - expectedWinRate)

actualResult:
승리 = 1
패배 = 0

K값:
기본 24 또는 32

전투 레이팅 랭킹의 장점:
- 유물 전투력이 상향평준화되어도 랭킹이 계속 변동됩니다.
- 단순히 유물을 많이 가진 사람이 아니라, 실제 배틀에서 성과를 낸 유저가 인정받습니다.
- 장기적으로 공동 1등 문제를 줄일 수 있습니다.

주의:
전투 레이팅은 유물 전투력과 다릅니다.

전투력 = 유물 기반 정적 힘
전투 레이팅 = 배틀 결과 기반 동적 경쟁 점수

━━━━━━━━━━━━━━━━━━━━
4. 배틀 승률 계산 개선
━━━━━━━━━━━━━━━━━━━━

배틀 승률은 실효 전투력을 기준으로 계산합니다.

추천 공식:

rawWinRate =
sigmoid((myEffectiveBattlePower - opponentEffectiveBattlePower) / BATTLE_SCALE)

최종 승률은 최소/최대값으로 제한합니다.

finalWinRate =
clamp(rawWinRate, MIN_WIN_RATE, MAX_WIN_RATE)

추천 기본값:

BATTLE_SCALE=2000
MIN_WIN_RATE=0.15
MAX_WIN_RATE=0.85

이렇게 하면 강한 유저가 대체로 유리하지만,
약한 유저도 일정 확률로 이길 수 있습니다.

기존처럼 강자/약자의 전투력 자체를 임의로 곱해서 보정하는 방식은 계산 결과가 직관적이지 않을 수 있으므로,
가능하면 clamp 방식으로 변경하는 것을 검토해주세요.

━━━━━━━━━━━━━━━━━━━━
5. 웹 랭킹 표시 방향
━━━━━━━━━━━━━━━━━━━━

웹 랭킹은 유저가 이해하기 쉬워야 합니다.

추천 탭 순서:

🌟 초능력자
💎 보유 포인트
🏺 유물 자산
⚔️ 전투력
🏆 시즌

각 탭의 의미:

🌟 초능력자:
보유 포인트 + 유물 자산가치 + 전투력 보너스를 반영한 대표 랭킹

💎 보유 포인트:
현재 가지고 있는 포인트 순위

🏺 유물 자산:
소유 유물 전체의 자산가치 순위

⚔️ 전투력:
실제 배틀에 적용되는 전투력 순위

🏆 시즌:
최근 활동성 또는 전투 레이팅 순위

각 유저 행에는 가능하면 다음 정보를 표시해주세요.

- 닉네임
- 대표 점수
- 보유 포인트
- 유물 자산가치
- 실효 전투력
- 최고 등급 유물
- 배틀 레이팅 또는 시즌 점수

━━━━━━━━━━━━━━━━━━━━
6. 구현 시 주의사항
━━━━━━━━━━━━━━━━━━━━

1. 포인트와 점수 단위를 구분해주세요.

포인트:
실제로 사용할 수 있는 재화
표기: P

랭킹 점수:
계산된 평가값
표기: 점

예시:
보유 포인트: 500,000P
유물 자산: 800,000점
초능력자 점수: 1,350,000점

2. users.score는 현재 보유 포인트인지, 누적 획득 포인트인지 명확히 해주세요.

가능하다면 다음 컬럼을 분리해주세요.

users.score = 현재 보유 포인트
users.total_earned_score = 누적 획득 포인트
users.total_spent_score = 누적 소비 포인트

3. 유물 자산가치는 실제 사용한 포인트 기준으로 계산해주세요.

강화 비용 절감, 이벤트 할인, 보너스 등이 적용된 경우,
명목 비용이 아니라 실제 지불 비용을 기준으로 누적해야 합니다.

4. 합성 유물의 자산가치 승계 로직을 반드시 설계해주세요.

합성으로 생성된 유물이 기존 재료 유물의 가치를 잃어버리면,
유물 자산 랭킹에서 유저가 손해를 보는 문제가 생깁니다.

5. 전투력과 유물 자산가치는 다른 개념입니다.

유물 자산가치:
얼마나 많이 투자했는가

실효 전투력:
실제로 얼마나 강한가

이 둘을 혼동하지 말고 별도 함수로 구현해주세요.

━━━━━━━━━━━━━━━━━━━━
7. 필요한 함수 예시
━━━━━━━━━━━━━━━━━━━━

다음 함수들을 구현하거나 기존 함수에서 분리해주세요.

1. calculateRelicAssetValue(relic)

개별 유물의 자산가치 계산

반환값:
number

2. calculateUserRelicAssetValue(userId)

유저가 소유한 모든 유물의 자산가치 합산

반환값:
number

3. calculateRelicPower(relic)

개별 유물 전투력 계산

반환값:
number

4. calculateEffectiveBattlePower(userId)

메인 유물 100% + 실제 적용 보관함 유물 35.31% 기준 실효 전투력 계산

반환값:
{
  totalPower,
  mainPower,
  storagePowerApplied,
  storageApplyRate
}

5. calculateSuperpowerScore(userId)

초능력자 대표 점수 계산

공식 예시:
보유 포인트 + 유물 자산가치 + 전투력 보너스

반환값:
{
  totalScore,
  pointScore,
  relicAssetScore,
  battlePowerBonus
}

6. calculateSeasonScore(userId, seasonId)

시즌 활동 점수 계산

반환값:
number

7. updateBattleRating(winnerId, loserId)

배틀 결과에 따른 레이팅 갱신

반환값:
{
  winnerOldRating,
  winnerNewRating,
  loserOldRating,
  loserNewRating
}

8. 최종 목표
━━━━━━━━━━━━━━━━━━━━

이번 개편의 최종 목표는 다음과 같습니다.

1. 포인트를 모으는 유저도 인정받게 한다.
2. 포인트를 유물에 투자한 유저도 손해 보지 않게 한다.
3. 실제로 강한 유저는 전투력 랭킹에서 인정받게 한다.
4. 장기적으로 유물 상향평준화가 발생해도 시즌/레이팅 랭킹을 통해 순위가 계속 움직이게 한다.
5. 유저가 랭킹을 봤을 때 “왜 내가 이 순위인지” 납득할 수 있게 한다.

따라서 단순히 기존 통합 랭킹 공식을 조금 수정하는 수준이 아니라,
포인트, 유물 자산, 실효 전투력, 시즌/레이팅을 분리한 구조로 설계해주세요.

다만 현재 코드베이스와 DB 구조를 고려하여,
한 번에 모든 기능을 구현하기 어렵다면 다음 순서로 단계적 구현을 제안해주세요.

1단계:
유물 자산가치 계산 추가

2단계:
실효 전투력 계산 추가

3단계:
랭킹 탭을 보유 포인트 / 유물 자산 / 전투력으로 분리

4단계:
초능력자 대표 랭킹 추가

5단계:
시즌 랭킹 또는 전투 레이팅 랭킹 추가

각 단계별로 필요한 DB 변경사항, 함수 변경사항, API 변경사항, 웹 UI 변경사항을 작업 단위로 나누어 제안해주세요.
