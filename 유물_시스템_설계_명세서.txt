이 문서는 "초능력자" KakaoTalk 봇의 유물 시스템 전체 파라미터,
그리고 조건부 마르코프 체인을 이용한 기댓값(E[X]) 계산 방식과
그 결과를 완전하게 기술합니다.

----------------------------------------------------------------
1. 시스템 개요
----------------------------------------------------------------

유물(Artifact)은 유저가 퀴즈 포인트를 소비해 강화하는 장비 시스템입니다.

- 성급: 1성 / 2성 / 3성 (3성이 최종)
- 강화 레벨:
    1성: 0강 ~ 50강 (최대 50강)
    2성: 0강 ~ 80강 (최대 80강)
    3성: 0강 ~ 100강 (최대 100강)
- 시작: 1성 1강으로 초기 생성
- 성급 승급 조건: 해당 성급 최대 레벨 달성 후 성급강화 시도 가능

----------------------------------------------------------------
2. 일반 강화 비용 (포인트)
----------------------------------------------------------------

강화 비용 = calcUpgradeCost(star, currentLevel)
즉, n강 → n+1강 시도 시 아래 비용이 차감됩니다.

[1성 강화 비용]
  구간          공식                    예시 값
  ──────────    ──────────────────      ───────
  n = 0강       0 P                     0P
  1 ≤ n ≤ 10   2 × n                   n=5: 10P / n=10: 20P
  11 ≤ n ≤ 20  20 + 2 × (n - 10)       n=15: 30P / n=20: 40P
  21 ≤ n ≤ 30  40 + 3 × (n - 20)       n=25: 55P / n=30: 70P
  31 ≤ n ≤ 40  70 + 5 × (n - 30)       n=35: 95P / n=40: 120P
  41 ≤ n ≤ 49  120 + 7 × (n - 40)      n=45: 155P / n=49: 183P

  대략적인 총합(0→50강 순방향 단순합): ~3,350P

[2성 강화 비용]
  구간          공식                    예시 값
  ──────────    ──────────────────      ───────
  0 ≤ n ≤ 40   15 + 3 × n              n=0: 15P / n=20: 75P / n=40: 135P
  41 ≤ n ≤ 60  135 + 6 × (n - 40)      n=50: 195P / n=60: 255P
  61 ≤ n ≤ 70  255 + 10 × (n - 60)     n=65: 305P / n=70: 355P
  71 ≤ n ≤ 79  355 + 15 × (n - 70)     n=75: 430P / n=79: 490P

  대략적인 총합(0→80강 순방향 단순합): ~14,000P

[3성 강화 비용]

3. 일반 강화 확률 (성공 / 유지 / 실패)
----------------------------------------------------------------

강화 1회 시도 결과는 3가지:
  성공(success): 레벨 +1 (n강 → n+1강)
  유지(keep):   레벨 변화 없음 (n강 → n강)
  실패(fail):   레벨 -1 (n강 → n-1강), 단 0강이면 0강 유지

(success + keep + fail = 100%)

  16 ~ 25강     76       16        8
  26 ~ 35강     70       19       11
  36 ~ 45강     65       21       14
  46 ~ 50강     60       23       17

4. 성급강화 (Promotion)
----------------------------------------------------------------

성급강화는 해당 성급 최대 레벨(1성=50강, 2성=80강) 달성 후
별도 명령어로 시도합니다.

[성급강화 비용]
  1성 → 2성 시도 비용:  190P
  2성 → 3성 시도 비용:  505P

[성급강화 성공률]
  1성 → 2성:  55% (성공)  /  45% (실패)
  2성 → 3성:  55% (성공)  /  45% (실패)

[성급강화 성공 시]
  - 다음 성급 0강으로 승급
  - 유물 이름 새로 부여

[성급강화 실패 시 (강등 레벨 랜덤 지급)]
  실패 시 현재 성급 내 특정 레벨로 강등되며, 레벨은 확률적으로 결정됩니다.

  1성 실패 → 1성 30~39강 중 랜덤 복귀:
    30강: 18%
    31강: 16%
    32강: 14%
    33강: 12%
    34강: 10%
    35강:  9%
    36강:  7%
    37강:  6%
    38강:  5%
    39강:  3%
    (합계 100%)

  2성 실패 → 2성 50~59강 중 랜덤 복귀 (동일 분포):
    50강: 18%
    51강: 16%
    52강: 14%
    53강: 12%
    54강: 10%
    55강:  9%
    56강:  7%
    57강:  6%
    58강:  5%
    59강:  3%
    (합계 100%)

  참고: 실패 시의 기댓 복귀 레벨
    1성 실패 기댓값 = 18×30 + 16×31 + ... + 3×39 / 100 ≈ 33.1강
    2성 실패 기댓값 = 18×50 + 16×51 + ... + 3×59 / 100 ≈ 53.1강

5. 기타 파라미터
----------------------------------------------------------------

[유물 보너스율 (퀴즈 포인트 획득 보정)]
  1성: min(25%, 5% + 0.4% × level)    → 50강: 25%
  2성: min(50%, 15% + 0.45% × level)  → 80강: 51% → 상한 50%
  3성: min(80%, 30% + 0.5% × level)   → 100강: 80%

[랜덤 콤보 발동률 (%)]:
  1성: min(5%,  1%  + 0.08% × level)
  2성: min(10%, 4%  + 0.08% × level)
  3성: min(20%, 8%  + 0.15% × level)

[일일 랜덤 콤보 최대 발동 횟수]:
  1성: 3회 / 2성: 5회 / 3성: 8회

[전투력(Power)]:
  1성: 100 + 5 × level      (50강: 350)
  2성: 450 + 10 × level     (80강: 1,250)
  3성: 1500 + 18 × level    (100강: 3,300)

[유물 교체 비용]:
  1성: 300P / 2성: 1,500P / 3성: 5,000P

[배틀 신청비]:
  1성: 100P / 2성: 500P / 3성: 1,500P

6. E[X] 계산 방법 - 조건부 마르코프 체인
----------------------------------------------------------------

목표: "현재 n강에서 최대강까지 올리는 데 기대되는 총 포인트 비용"을
      E[X]로 정의하고, 모든 확률 변수의 조건부 연관성을 반영하여 계산.

[핵심 아이디어]

유물 강화는 현재 레벨(상태)에만 의존하는 마르코프 체인(Markov Chain)입니다.
즉, 다음 상태의 전이 확률은 현재 상태에만 조건부로 결정됩니다.

T[n] = "현재 n강일 때, n강 → n+1강으로 올리는 데 드는 기댓 포인트 비용"

[T[n] 점화식 (재귀)]

  n = 0강일 때:
    T[0] = c(0) / p_s
    (실패해도 0강 유지이므로 T[-1] = 0)

  n > 0강일 때:
    T[n] = (c(n) + p_f × T[n-1]) / p_s

  여기서:
    c(n)  = n강에서 강화 1회 시도 비용 (포인트)
    p_s   = 성공 확률 (소수점, 예: 0.87)
    p_k   = 유지 확률 (keep)
    p_f   = 실패 확률
    T[n-1] = n-1강에서 n강까지 올리는 기댓 비용 (이미 계산된 값)

  유도 과정:
    T[n] = c(n) + p_s×0 + p_k×T[n] + p_f×(T[n-1] + T[n])
    T[n]×(1 - p_k - p_f) = c(n) + p_f×T[n-1]
    T[n]×p_s = c(n) + p_f×T[n-1]
    → T[n] = (c(n) + p_f × T[n-1]) / p_s

[왜 독립이 아닌 조건부인가?]

  T[n] 식 안에 T[n-1]이 포함되어 있습니다.
  이는 "n강에서 실패할 경우 n-1강으로 돌아가고,
  다시 n-1강 → n강을 올리는 비용(T[n-1])을 추가로 지불해야 함"을
  재귀적으로 반영한 것입니다.

  따라서 각 강화 시도는 독립적인 베르누이 시행이지만,
  강화 비용 전체는 이전 상태에 조건부로 연결된 구조입니다.

[전체 기댓 비용]

  0강에서 최대강(N강)까지의 총 기댓 비용:
    E[0 → N강] = Σ T[n],  n = 0, 1, 2, ..., N-1

[성급강화 기댓 비용]

  성급강화 실패 시 특정 레벨 k로 복귀 (확률 분포 P(복귀=k)):

  E[실패 후 복귀 비용] = Σ_k P(복귀=k) × Σ_{n=k}^{N-1} T[n]
  (k강에서 다시 N강까지 재강화하는 비용의 기댓값)

  E[성급강화] = (성급비용 + P(실패) × E[실패 후 복귀 비용]) / P(성공)
