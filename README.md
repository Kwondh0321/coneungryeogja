## 1. 게임 시스템

### 🔤 초성퀴즈

화면에 표시된 초성(자음)을 보고 단어를 맞히는 게임.

| 구분 | 포인트 |
|---|---|
| 랜덤 문제 정답 | 최대 **7,500P** |
| 주제 선택 정답 | 최대 **3,750P** |
| 오답 시 | 감점 누적 (최소 50P 보장) |
| 힌트 사용 | −100P |

- 5회 이상 오답 → 글자 힌트 자동 공개
- 60초 내 미정답 → 자동 종료
- 명령어: `@초능력자 초성퀴즈`, `@초능력자 초성퀴즈 [카테고리]`, `힌트`, `다음문제`, `종료`

---

### 📜 훈민정음 (멀티플레이)

초성 + 글자 수 힌트로 단어를 맞히는 실시간 멀티플레이 게임.

| 구분 | 포인트 |
|---|---|
| 단어 1개 정답 | **2,500P** |
| 최다 정답자 MVP | +**12,000P** 보너스 |

- 방 개설 후 25초 이내 `참여` 입력으로 입장
- 45초 동안 최대한 많은 단어를 맞히면 승리
- 이미 맞힌 단어 재사용 불가
- 명령어: `@초능력자 훈민정음`, `참여`, `시작`, `힌트`, `종료`

---

### 🔡 자모연성

한글 자모 조각을 보고 4지선다로 단어를 맞히는 게임.

| 난이도 | 글자 수 | 기본 포인트 |
|---|---|---|
| 쉬움 | 2글자 | **1,780P** |
| 보통 | 3글자 | **2,670P** |
| 어려움 | 4글자 | **4,000P** |

**추가 보너스:**

| 조건 | 보너스 |
|---|---|
| 3글자 이상 단어 | +670P |
| 받침 포함 | +440P |
| 오늘 첫 자모연성 | +1,110P |
| 연속 3회 정답 | +2,220P |

- 3분 이내 미응답 시 자동 종료
- 명령어: `@초능력자 자모연성 [쉬움/보통/어려움]`, `1~4`, `다음퀴즈`, `자모포기`

## 2. 콤보 시스템

```
10분 내 게임 5회 승리 → 콤보 발동!
→ 60초간 모든 게임 포인트 +20%
```

- 게임 종류 무관 (초성퀴즈 / 훈민정음 / 자모연성 통합)
- 유저 개인 독립 콤보 (다른 유저와 간섭 없음)
- 콤보 발동 중 카운터 정지 → 만료 후 재카운트 시작
- `computeReward()` 함수에서 콤보 + 유물 보너스 통합 계산

## 3. 포인트 & 랭킹

### 포인트 (P)

- `users.score` 컬럼에 저장
- 게임 승리 / 콤보 / 유물 보너스 등으로 획득
- 유물 강화·레벨업·합성·뽑기 등에 소비

### 랭킹

**초능력자 점수** = 포인트 + 유물 자산가치 + 전투력을 합산한 통합 순위

- `/api/ranking` — 서버·방 별 랭킹 조회
- 웹사이트 `chosung.app/ranking` 에서 실시간 확인

### 기타

- `출석체크 / 출첵 / ㅊㅊ` — 일일 출석 보상
- `선물 @닉네임 N` — 일일 5회, 최대 30,000P 선물 가능

## 4. 유물 시스템

`artifacts/api-server/src/lib/newRelicSystem.ts` 에 전체 로직 포함.

### 등급 체계

| 등급 | 기호 | 기본 효과값 | 획득 방법 |
|---|---|---|---|
| D | ⬜ | 6.45% | 드랍 (4.41%), 뽑기 |
| C | 🟩 | 9.82% | 드랍 (3.50%), 합성 |
| B | 🟦 | 16.43% | 드랍 (0.48%), 합성 |
| A | 🟪 | 28.29% | 드랍 (0.08%), 합성 |
| S | 🟨 | 45.06% | **합성 전용** |
| SS | 🔴 | 95.0% | **합성 전용 (신화)** |

> S·SS는 드랍 없음, 오직 합성으로만 획득 가능.

### 유물 타입 (8종)

| 타입 | 효과 |
|---|---|
| 🧭 나침반 | 모든 게임 포인트 +X% |
| ⏳ 모래시계 | 유물 강화 비용 −X% |
| 🔦 랜턴 | 초성퀴즈 포인트 +X% |
| 🪞 거울 | 훈민정음 포인트 +X% |
| 💰 보물함 | 자모연성 포인트 +X% |
| 👑 왕관 | 연속 정답 보너스 +X% |
| 🧪 물약 | 보관함 유물 효과 +X% |
| 🪶 깃털펜 | 유물 경험치 획득 +X% |

### 효과값 계산 공식

```
effectValue = BASE_EFFECT[grade] × (1 + (level−1) × 0.01) × (1 + enhance × 0.03)
```

| 파라미터 | 범위 | 설명 |
|---|---|---|
| level | 1 ~ 50 | 레벨 1당 효과 +1% |
| enhance | +0 ~ +20 | 강화 1단계당 효과 +3% |

**SS 이중 효과 (신화 전용)**  
메인 또는 보관함에 SS 유물이 있으면 `all_bonus`가 추가 적용됨:
```
allBonus += effectValue × 0.50
```

### 보관함

- 메인 유물 1개 + 보관함 유물 N개 (기본 슬롯, 확장 가능)
- 보관함 유물은 메인 효과의 **20%** 적용
- 같은 효과 타입 최대 3개까지 중첩

### 강화

| 구간 | 성공률 예시 |
|---|---|
| +0 → +1 | 95.00% |
| +5 → +6 | 73.08% |
| +10 → +11 | 36.42% |
| +19 → +20 | 8.52% |

- 강화 실패 시 파괴 없음 (단계 유지 또는 하락)
- 비용: `grade × 568 × 1.1166^enhance`

### 합성 성공률

**일반 합성 (D → C → B → A → S)**

성공률은 재료 수·강화 수치·등급으로 결정되며 최대 95%.

| 재료 → 결과 | 비용 |
|---|---|
| D → C | 3,185P |
| C → B | 9,782P |
| B → A | 32,528P |
| A → S | 130,335P |
| S → SS | 500,000P |

**SS 합성**

```
최소 S등급 유물 3개 이상
성공률 최대 46%
```

### 분해 환급

| 등급 | 기본 환급 |
|---|---|
| D | 630P |
| C | 2,530P |
| B | 10,100P |
| A | 37,970P |
| S | 126,570P |
| SS | 400,000P |

강화·레벨업에 소비된 비용의 일부도 추가 환급.

## 5. 배틀 시스템

`artifacts/api-server/src/lib/battleSystem.ts`

### 전투력 계산

```
전투력 = f(포인트, 유물 effectValue, 유물 등급)
```

- 전투력이 높은 유저가 배틀에서 유리
- SS 유물 보유 시 전투력 대폭 상승

### 승률 범위

| 조건 | 최소 승률 |
|---|---|
| 기본 | 5% |
| 상대가 SS 보관함만 보유 | 3% |
| 상대가 SS 메인만 보유 | 2% |
| 상대가 SS 메인+보관함 모두 | 1% |
| 최대 승률 (공통) | 95% |

### 보상

```
승리 보상 = 상대 전투력 × 7  (최대 3,000,000P)
```

## 8. 기술 스택 & 아키텍처

```
chosung-app/
├── artifacts/
│   ├── api-server/          # Express 5 API 서버
│   │   └── src/
│   │       ├── lib/         # 게임 로직 (gameState, newRelicSystem, battleSystem, jamoGame ...)
│   │       └── routes/      # 라우트 (kakao, admin, profile, health ...)
│   └── chosung-quiz/        # React + Vite 웹사이트
│       └── src/
│           ├── ProfilePage.tsx
│           ├── RankingPage.tsx
│           ├── DetailPage.tsx
│           └── PatchNotesPage.tsx
├── lib/                     # 공유 라이브러리 (db, api-spec ...)
└── scripts/                 # 유틸리티 스크립트
```

| 영역 | 스택 |
|---|---|
| 런타임 | Node.js 24 |
| 언어 | TypeScript 5.9 (strict) |
| 패키지 관리 | pnpm workspaces |
| API 서버 | Express 5 |
| DB | PostgreSQL + Drizzle ORM |
| 유효성 검사 | Zod (v4) + drizzle-zod |
| API 코드젠 | Orval (OpenAPI → React Query hooks + Zod) |
| 프론트엔드 | React + Vite |
| 빌드 | esbuild (CJS bundle) |
| 로깅 | Pino |

**서비스 라우팅**: Replit 역방향 프록시가 경로 기반으로 서비스를 라우팅.  
- `/api/*` → `api-server` (포트 8080)  
- `/*` → `chosung-quiz` (포트 $PORT)

## 9. 개발 환경 설정

```bash
# 의존성 설치
pnpm install

# API 서버 실행
pnpm --filter @workspace/api-server run dev

# 웹사이트 실행
pnpm --filter @workspace/chosung-quiz run dev

# 전체 타입체크
pnpm run typecheck

# DB 스키마 푸시 (개발용)
pnpm --filter @workspace/db run push

# OpenAPI → 클라이언트 코드 재생성
pnpm --filter @workspace/api-spec run codegen

# 전체 빌드
pnpm run build
```
